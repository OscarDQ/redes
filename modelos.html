Modelos de Red

Modelos de Referencia: OSI y TCP/IP

Para que millones de dispositivos de distintos fabricantes puedan comunicarse a través de Internet, es necesario establecer un conjunto de reglas y estándares universales.

Propósito: Los modelos de red organizan y estandarizan la transmisión de datos dividiéndola en capas independientes, facilitando el diagnóstico de fallos y la interoperabilidad entre fabricantes.

El Concepto General: Encapsulamiento (PDU)

El encapsulamiento es el proceso mediante el cual cada capa añade una cabecera de control al mensaje antes de pasarlo a la capa inferior, hasta convertirlo en señales físicas transmisibles.

Segmento

El mensaje se divide en fragmentos y se asignan los puertos lógicos de origen y destino.

Paquete

Se añaden las direcciones IP de origen y destino para el enrutamiento global.

Trama (Frame)

Se incorpora la dirección MAC física del dispositivo para la entrega local.

Pulsos Puros

El conjunto de datos se convierte en impulsos eléctricos o ópticos transmitidos por el medio físico.

Modelo OSI

El Estándar Teórico Universal de Interconexión

¿Qué es?

Marco conceptual creado por la ISO en 1984 para estandarizar la intercomunicación entre dispositivos de distintos fabricantes.

¿Qué hace?

Divide la transmisión de datos en siete capas independientes, cada una con una función específica y bien definida.

Ventaja Clave

El aislamiento por capas permite identificar y corregir fallos sin afectar el resto del sistema.

Regla mnemotécnica (de abajo a arriba): "FERTSPA" → Física, Enlace, Red, Transporte, Sesión, Presentación, Aplicación. Memoriza este acrónimo para dominar el modelo OSI de inmediato.

7

APLICACIÓN — Interfaz Humana

Constituye la interfaz directa entre las aplicaciones del usuario y la red. Permite a los programas acceder a los servicios de red.

Ejemplos: HTTP, FTP, SMTP, DNS

6

PRESENTACIÓN — Traductor Criptográfico

Garantiza la compatibilidad del formato de datos entre sistemas distintos. Gestiona el cifrado, la compresión y la conversión de formatos.

Ejemplos: SSL, TLS, JPEG

5

SESIÓN — Moderador Inmortal

Establece, mantiene y finaliza las sesiones de comunicación entre aplicaciones, gestionando puntos de control para reanudar transferencias interrumpidas.

Ejemplos: NetBIOS, PPTP

4

TRANSPORTE — Logística a Escala

Gestiona el control de flujo y la fiabilidad de la entrega, segmentando los datos y verificando su recepción correcta.

TCP (Confiable): Exige acuse de recibo por cada segmento entregado. Si no hay confirmación (ACK), reenvía el dato. Lento pero con confianza 100%. Usado en correo, descarga de archivos, webpages.
UDP (Veloz): Dispara los fragmentos sin confirmar recepción ni orden de llegada. Indispensable donde la latencia es crítica: videollamadas Zoom, disparos en videojuegos multijugador, streaming en vivo.

Ejemplos: Protocolos TCP, UDP

3

RED — GPS del Planeta

Realiza el direccionamiento lógico mediante direcciones IP y calcula la ruta más eficiente para el envío de paquetes a través de la red global.

Ejemplos: Direcciones IP, Routers, BGP

2

ENLACE DE DATOS — Agente Físico

Gestiona la transferencia confiable de datos entre nodos adyacentes mediante tramas, detectándolas colisiones y garantizando la entrega en la red local.

Ejemplos: MAC Address, Switches Ethernet

1

FÍSICA — Cruda Maquinaria

Define los medios físicos de transmisión, convirtiendo los datos en señales eléctricas, ópticas o electromagnéticas según el tipo de canal.

Ejemplos: Cables UTP, Fibra Óptica, Hubs

Modelo Pragmático TCP/IP: La Ingeniería Militar

El modelo TCP/IP es el protocolo de comunicaciones vigente en Internet. Fue desarrollado por encargo del Departamento de Defensa de los Estados Unidos (DARPA) durante la Guerra Fría, priorizando la resiliencia sobre la elegancia formal.


Filosofía de supervivencia: El sistema fue diseñado para mantener la comunicación activa incluso ante la destrucción parcial de la infraestructura, condensando las funciones de red en cuatro capas operativas robustas.

Capa 4: Aplicación Neta

Agrupa las funciones de Aplicación, Presentación y Sesión del modelo OSI en una sola capa, donde operan los protocolos de la Web y los servicios de red (HTTP, HTTPS, FTP, DNS, SMTP).

Capa 3: Transporte — Three-Way Handshake

Misma función que la Capa 4 de OSI. Aquí ocurre el famoso Three-Way Handshake de TCP: (1) Cliente: "Hola, ¿me escuchas?" → (2) Servidor: "Sí, habla." → (3) Cliente: "Perfecto, allá voy." Este apretón de manos de tres vías garantiza que la conexión está lista antes de enviar cualquier dato.

Capa 2: Internet (IP)

Corazón de TCP/IP. Opera mediante el protocolo IP, dictando si la comunicación usa la clásica IPv4 (direcciones de 32 bits, ~4 mil millones de direcciones) o la ilimitada IPv6 (128 bits). Incluye comandos como ping (ICMP) para verificar si un nodo remoto está activo.

Capa 1: Acceso Hostil o Físico Submarino

Integra las funciones del Enlace de Datos y la Capa Física del OSI. Controla el acceso al medio físico de transmisión, ya sea cableado o inalámbrico.

El brillante Tratamiento en Diagnósticos por Errores

El modelo OSI exigía verificaciones de errores en cada capa, lo que resultaba en mayor latencia. TCP/IP simplificó este proceso: la verificación de integridad se realiza únicamente en el destino final, eliminando comprobaciones intermedias. Esta decisión fue determinante para lograr la velocidad y escalabilidad que caracterizan al Internet actual.
